* BruceBase parser
** plan

The BruceBase website at http://brucebase.wikidot.com/ renders everything related to Bruce Springsteen in a highly
structured but HTML generated oriented way.

The goal of this userscript is, it will parse different aspects from the underlying HTML data and checks for
inconsistencies, by enriching the standard BrueBase pages with glyphs which shows discrepancies between different parts
of the website.

We will start with the generic YEAR event pages and check if the corresponding event DETAILS pages match with their
event names.

The URL structure for YEAR pages is

#+begin_example
http://brucebase.wikidot.com/<4 digit year>
#+end_example

e.g. "http://brucebase.wikidot.com/2024"

Typical event name looks like (in all UPPERCAE and a "-" between the date and venue) and hyperlinked to their event
DETAIL page (Capitalized and "The" inside "(" and ") before the ",") in

| YEAR page (event name)                             | Anchor in YEAR page   | Link                                                                          | DETAIL page (event name)                                       |
|----------------------------------------------------+-----------------------+-------------------------------------------------------------------------------+----------------------------------------------------|
| 2024-01-07 - THE BEVERLY HILTON, BEVERLY HILLS, CA | <a name="070124"></a> | http://brucebase.wikidot.com/nogig:2024-01-07-beverly-hilton-beverly-hills-ca | 2024-01-07 Beverly Hilton (The), Beverly Hills, CA |
| 2024-01-19 - FIVE RINGS FARM, WELLINGTON, FL       | <a name="190124"></a> | http://brucebase.wikidot.com/gig:2024-01-19-five-rings-farm-wellington-fl     | 2024-01-19 Five Rings Farm, Wellington, FL         |
|                                                    |                       |                                                                               |                                                    |

Write a userscript which parses the YEAR pages, and for each event follows the link to the detail page and compares the
event names. For the comparison to succeed

- the "(The)" in front of a "," should be treated as being right after the date (which additionally should get a " - " added)
 comparison should be made by UPCASING the DETAIL page event name
 
So e.g the two event names in the above table should be considered the same. In this case append a green checkmark after
the event name on the YEAR page and a red cross if they differ.

In both cases when hovering over the YEAR page event name, display in a rich HTML based tooltip the differences (if any)
in the event name (except if they differ in what we already discussed)

** setlist check

Add the functionality for "setlist/recording session" checks:

1.)

On YEAR pages for each event parse the "setlist/recording session" in an array of SONGS.

Setlists have the following typical structure @debug/YEAR.org

- songs have the string " / " between each other and are located inside a "<p>" container after the event name
- every song is UPPERCASED
- the HTML <strong> attribute should be ignored
- optional additional musicians after the UPPERCASED song name should be ignored e.g. "(with Willie Nile and Jon Landau)"
- sometimes an event has an additional "<p>" container which starts with a capitalized string like e.g. "Soundcheck:" and has the setlist after it with the same charactertistics as above

Recording sessions have the following typical structure:

http://brucebase.wikidot.com/1984#120184

#+begin_example
<blockquote>
<p>MAN AT THE TOP / ROCKAWAY THE DAYS</p>
</blockquote>
#+end_example

- the "<p>" container is addionally surrounded with a "<blockquote>" right after the event name

2.)

For each event follow the link to the detail page and again parse the "setlist/recording session" in an array of SONGS.

DETAIL/RECORDING SESSIONS have the following typical structure @debug/DETAIL.org

- songs are typically located inside a '<div id="wiki-tab-0-1"' container in an ordered list "<ol>" or unordered list "<ul>" in case of e.g. "Soundcheck" (see below)
- each song is again UPPERCASED and hyperlinked, only the UPPERCASR TEXT part should be taken into account
- sometimes when in the YEAR page the setlist starts with a capitalized string like e.g. "Soundcheck:", the DETAIL page 
  has '<p><strong>Soundcheck</strong></p>' in front of the unordered list. In such a case the ordered setlist is typically preceded with '<p><strong>Show</strong></p>'  

3.)

For each event compare the two song arrays and when on the YEAR page

- render a song title in "green" when they both match
- render differing chars in a song title in RED, show a rich HTML based tooltip where both versions show up
- render a missing song on the YEAR page (when it's additionally on the DETAIL page) with a background color like "yellow"
- render an additional song on the YEAR page (when it's missing on the DETAIL page) with a background color like "light blue"
- for the last two cases show a rich HTML tooltip on the songs explaining the fact

4.)

For each event compare the two song arrays and when on the DETAIL page, do the same as on the YEAR page but the other way around

5.) Read more examples if it's not yet clear what I mean
